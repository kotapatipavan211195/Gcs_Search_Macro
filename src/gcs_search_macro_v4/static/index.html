GCS Search Macro

Checking your corporate identity…

Add one row for each GCS bucket and optional path. The selected access profile controls which bucket/path roots are permitted.

Literal looks for the text exactly as entered, such as PID. Safe regex matches a pattern, such as pid_\d+. Groups, backreferences, and counted repetitions are intentionally disabled.

Filename search checks the final component of every blob name, regardless of extension. A complete filename match is exact; a filename containing the entered text is partial.

The destination must stay within an approved copy bucket and path root. Each job writes beneath its own folder to avoid collisions.

Use commas or semicolons for up to five recipients. Delivery is limited to approved organization email domains.
